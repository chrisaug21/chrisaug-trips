Trips Website Style Guide

A comprehensive guide to styling components across the travel site

Color Palette

Primary Colors

  • Primary: #0077CC - Used for headings, links, and key elements
  • Secondary: #8DA2C0- Used for secondary elements and accents
  • Accent: #F2C94C - Used for highlights, calls to action, and important elements

Neutral Colors

  • Dark Text: #333333 - Used for main body text
  • Light Text: #666666 - Used for secondary text
  • Light Background: #f8f9fa - Used for content backgrounds
  • White: #ffffff - Used for card backgrounds, text on dark backgrounds

Typography

Fonts

  • Headings: Nunito (300, 400, 600, 700)
  • Body: Lato (300, 400, 700)

For consistency, always use the defined fonts and weights. These are loaded via Google Fonts.

Heading Styles

H1: Page Title (2.5rem)

H2: Section Title (2rem)

H3: Content Heading (1.3rem)

H4: Subsection (1.2rem)

Text Styles

Regular paragraph text (1rem, Lato). This is used for the main body content throughout the site.

Strong text is used for emphasis and important information, typically colored in the primary color.

Italic text is used for quotes, tips, and to provide emphasis within a paragraph.

Links use the primary color and transition to secondary color on hover.

Navigation Components

Primary Navigation (Site Header)

The primary navigation appears at the top of every page and includes the site title and main navigation links. This is implemented using a component-based approach to ensure consistency across all pages.

Example

Component-Based Implementation

The site uses a component-based approach for navigation rather than hard-coding the navigation on each page:

  • Navigation is loaded dynamically using JavaScript components
  • The header-container element is populated by components.js
  • This ensures consistent navigation across all pages without duplication
  • Updates to navigation only need to be made in one place
// HTML placeholder in each page
<div id="header-container"></div>

// JavaScript in components.js loads the header dynamically
document.addEventListener('DOMContentLoaded', function() {
  // Load header component
  const headerContainer = document.getElementById('header-container');
  if (headerContainer) {
    headerContainer.innerHTML = `
      <header>
        <div class="container site-header">
          <h1 class="site-title"><a href="index.html">Trip Guides</a></h1>
          <button class="mobile-nav-toggle"><i class="fas fa-bars"></i></button>
          <nav>
            <ul>
              <li><a href="index.html">Home</a></li>
            </ul>
          </nav>
        </div>
      </header>
    `;
  }
});

Trip-specific Navigation

The trip-specific navigation appears below the main navigation on trip pages and provides navigation to specific days of the trip. Like the primary navigation, this is also implemented using a component-based approach.

Example
Trip Home Day 1 Day 2 Day 3

Component-Based Implementation

The trip-specific navigation is implemented as a component that's dynamically loaded on each page:

  • Navigation is loaded into the trip-nav-container element
  • The component script automatically determines the current page and highlights the active day
  • This approach ensures consistent navigation across all trip pages
  • Changes to navigation structure only need to be made in one place
// HTML placeholder in each page
<div id="trip-nav-container"></div>

// JavaScript in components.js loads the trip navigation dynamically
function loadTripNav() {
  const tripNavContainer = document.getElementById('trip-nav-container');
  if (tripNavContainer) {
    // Determine current page for active highlighting
    const currentPage = window.location.pathname.split('/').pop();
    
    // Generate day links with appropriate active class
    let dayLinks = '';
    for (let i = 1; i <= 11; i++) {
      const isActive = currentPage === `day${i}.html` ? 'active' : '';
      dayLinks += `<a href="day${i}.html" class="trip-nav-link ${isActive}">Day ${i}</a>`;
    }
    
    tripNavContainer.innerHTML = `
      <div class="trip-nav">
        <div class="container">
          <div class="trip-nav-container">
            <div class="trip-nav-days">
              <a href="index.html" class="trip-nav-link trip-home-link">
                <i class="fas fa-home"></i> <span>Trip Home</span>
              </a>
              ${dayLinks}
            </div>
          </div>
        </div>
      </div>
    `;
  }
}

Implementation Notes

  • Always include a "Trip Home" link that returns to the trip's main page
  • Highlight the active day using the .active class
  • The navigation should be sticky at the top of the screen during scrolling
  • On mobile, the navigation collapses to a grid of buttons
  • Never hard-code the navigation directly in each page's HTML

Lower Navigation

Lower navigation buttons appear at the bottom of day pages to allow users to move between consecutive days.

Example
Previous: Day 1 Next: Day 3

Implementation Notes

  • Always include both Previous and Next buttons for consistent navigation
  • The Previous button should be blue (primary color)
  • The Next button should be yellow (accent color)
  • Button Text Format: Always use the exact format "Previous: Day X" and "Next: Day X" where X is the day number (e.g., "Previous: Day 2", "Next: Day 3")
  • Do NOT use descriptive day names (like "Previous: Travel Day") or any other format
  • For the first day, still include a Previous button that links to the trip home page with text "Previous: Trip Home"
  • For the last day, still include a Next button that links to the trip home page with text "Next: Trip Home"
<div class="navigation-buttons">
  <a href="day1.html" class="btn">Previous: Day 1</a>
  <a href="day3.html" class="btn btn-accent">Next: Day 3</a>
</div>

Remember to add these navigation buttons to all day pages, including day1.html which is currently missing them.

Content Components

Content Component Overview

There are several distinct content components designed for different types of information. Each has a specific purpose and visual style to help users identify the type of content.

Cultural Box (Yellow Left Border)

Cultural Box

This component is used for general information, cultural context, and descriptive content. It features a yellow/accent-colored left border with a light background.

Use this component for:

  • Historical information
  • Cultural context
  • General descriptions
  • Background information
Travel Detail (Blue Left Border)

Travel Detail

This component is used for key travel logistics, schedules, and itinerary information. It features a blue/primary-colored left border with a light background.

Use this component for:

  • Transportation details
  • Accommodation information
  • Schedules and timetables
  • Logistical information

Flexible Layout Components

The following components can be used flexibly in layouts. They can be full-width or arranged side-by-side to avoid creating unnecessary white space.

IMPORTANT: Option boxes, location highlights, and other flexible components should be placed side by side at half-width whenever possible, rather than stacked full-width. This creates more visually balanced layouts, reduces excessive scrolling, and makes better use of screen space on larger displays. Reserve full-width layouts for when the content truly requires the additional space.

Location Highlight (Yellow Background)

Location Highlight

This component is used to highlight specific locations, routes, or attractions. It features a light yellow/accent background with a yellow/accent-colored left border.

Options Box (Blue Background)

Options Box

This component is used for alternative options, evening activities, or secondary information. It features a light blue/primary background with a blue/secondary-colored left border.

Historical and Cultural Context Styling

IMPORTANT: All historical and cultural context information should be visually distinct with a background color that contrasts with its surrounding container. These elements can be nested inside other components, but must use a different background color than their parent container.

For example, if historical context is nested inside a travel-detail box (blue border with white background), the historical context should use a location-highlight style (yellow background). If nested inside a component with a yellow background, consider using a light blue background instead.

❌ Incorrect Implementation

Never use the same background color for nested historical context as its parent container:

<div class="travel-detail">
  <h3>Castle Information</h3>
  <p>Practical information...</p>
  
  <div class="culture-box"> <!-- Same white background as parent -->
    <h3>Historical Context</h3>
    <p>Historical information...</p>
  </div>
</div>

✅ Correct Implementation

Use contrasting background colors for nested components:

<!-- Option 1: Separate components -->
<div class="travel-detail">
  <h3>Castle Information</h3>
  <p>Practical information...</p>
</div>

<div class="location-highlight">
  <h3>Historical Context</h3>
  <p>Historical information...</p>
</div>

<!-- Option 2: Nested with contrasting background -->
<div class="travel-detail">
  <h3>Castle Information</h3>
  <p>Practical information...</p>
  
  <div class="location-highlight"> <!-- Yellow background contrasts with parent -->
    <h3>Historical Context</h3>
    <p>Historical information...</p>
  </div>
</div>

The key is visual distinction - if you're nesting content components, always ensure they have different background colors to clearly separate the content types for users.

Option Tabs for Multiple Day Options

Purpose and Usage

Option tabs are used on days that offer multiple alternative itineraries (e.g., "Option Day" where travelers can choose between different activities or destinations). This component allows users to switch between different day plans without leaving the page.

The option tabs system consists of two main components:

  1. Tab Navigation - A row of tabs at the top that users click to switch between options
  2. Tab Content - The corresponding content sections that are shown/hidden based on which tab is active
Option Tabs Example
Option A
Option B

This is the content for Option B. Users can click between tabs to view different day options without leaving the page.

Tab Labeling Guidelines

IMPORTANT: Tabs should be labeled simply with "Option A", "Option B", etc., rather than including subtitles or descriptions directly on the tab. The title of the theme or concept for each option should then be listed at the top of the content for that option. This approach:

  • Saves valuable space in the navigation area, especially on mobile devices
  • Creates a cleaner, more consistent interface
  • Improves readability and click targets on smaller screens
  • Allows for better responsiveness across different device sizes

❌ Incorrect Tab Labels

<div class="option-tabs">
  <div class="option-tab active">Option A: Full Day in Amsterdam with Museum Visits</div>
  <div class="option-tab">Option B: Day Trip to Haarlem and Local Sightseeing</div>
</div>

✅ Correct Tab Labels

<div class="option-tabs">
  <div class="option-tab active">Option A</div>
  <div class="option-tab">Option B</div>
</div>

Then include a descriptive heading within each option's content area:

<div class="option-content active" id="option-a">
  <div class="section-title-container">
    <h2>Full Day in Amsterdam with Museum Visits</h2>
  </div>
  <!-- Rest of option A content -->
</div>

You can include a brief description of the options directly beneath the tabs in a small introduction paragraph to help users choose between options.

JavaScript Functionality

document.addEventListener('DOMContentLoaded', function() {
  // Option tabs functionality
  const optionTabs = document.querySelectorAll('.option-tab');
  const optionContents = document.querySelectorAll('.option-content');
  
  optionTabs.forEach(tab => {
    tab.addEventListener('click', () => {
      // Remove active class from all tabs and contents
      optionTabs.forEach(t => t.classList.remove('active'));
      optionContents.forEach(c => c.classList.remove('active'));
      
      // Add active class to clicked tab
      tab.classList.add('active');
      
      // Show corresponding content
      const targetId = tab.dataset.option;
      const targetContent = document.getElementById(targetId);
      if (targetContent) {
        targetContent.classList.add('active');
      }
    });
  });
});

Responsive Behavior

The tab system is responsive by design:

  • On desktop, tabs appear side by side in a horizontal row
  • On mobile, tabs maintain their horizontal layout but use more compact padding
  • Each tab content area contains its own responsive components

The option tabs should always remain visible at the top of the page, with content scrolling below.

Mobile Responsiveness Guidelines

General Principles

All components and layouts should be responsive and function well on mobile devices. The site uses a mobile-first approach, with layouts that adapt to larger screens.

  • Breakpoint: The main responsive breakpoint is 768px, below which mobile styles are applied
  • Container width: Use percentage-based widths with max-width constraints
  • Typography: Font sizes should remain readable on small screens

Header & Navigation

  • Primary navigation collapses to a hamburger menu on mobile
  • Trip day navigation transforms to a grid layout on mobile
  • The "Trip Home" link text changes to just "Trip" on mobile
  • Navigation buttons become full-width and stack vertically

Content Components

  • Content grids collapse to a single column on mobile
  • Padding and margins are reduced on mobile for better space usage
  • Content boxes maintain the same styling but adjust width
  • Font sizes may be slightly reduced on very small screens

Mobile Testing Guidelines

  • Test all pages on multiple device sizes
  • Ensure all text is readable without zooming
  • Verify that touch targets (buttons, links) are large enough (minimum 44x44px)
  • Check that navigation remains usable and intuitive
  • Test on both iOS and Android devices when possible
  • Use Chrome DevTools device emulation for initial testing

Always test new pages and components on actual mobile devices or using responsive design mode in your browser's developer tools.

Implementation Tips

General Guidelines

  • Consistency: Maintain consistent styling across all pages
  • Flexibility: Choose appropriate components based on content type
  • Mobile-first: Ensure all components work well on mobile devices
  • Visual hierarchy: Use styling to guide the user's attention
  • Side-by-side layouts: Prioritize placing flexible components side by side over stacking them full-width

Not every page will need all component types. Choose the most appropriate components for your specific content.

Component Selection Guide

  • Cultural Box: General information, cultural context, descriptions
  • Travel Detail: Logistical information, schedules, itineraries
  • Location Highlight: Key attractions, points of interest, routes
  • Options Box: Alternative options, evening activities, additional information
  • Travel Tip: Advice, insider information, helpful hints
